System / Backend & Data Model

Backend & Data Model

Architecture

The Excel workbook splits into three layers: raw facts land via imports, business rules live in SQL views + a FastAPI service, and this web UI replaces the sheets.
SourcesWix POS export · Forecasting Tool (Office Scripts push) · manual weekly inputs
→
Import layerValidation, name mapping, dedupe, 3-grain cross-check · import_runs audit
→
PostgreSQLMaster data + fact tables · calculated fields as SQL views (no cell formulas)
→
FastAPI backendREST endpoints, auth, Excel export, recalculation jobs
→
Web frontendDashboard · weekly entry · monthly P&L · master data · import review
What this buys over the workbook: months are rows, not pasted column blocks — so “compare April 2026 vs April 2025” is a query, not a copy-paste; settings are versioned so old weeks keep old rates; name mismatches surface as tasks; and every number traces back to an import run.

Database schema

green = master data (dimensions) · blue = facts & inputs · amber = calculated SQL views

fridges

  • id serial PK
  • name text unique
  • yearly_fee numeric
  • contract_end_date date
  • is_active bool

suppliers

  • id serial PK
  • name text unique
  • aliases text[]

categories

  • id serial PK
  • name text unique
  • sort_order int

settings

  • id serial PK
  • key text
  • value numeric/text
  • effective_from date

weeks

  • id serial PK
  • year, week_no int, unique
  • start_date, end_date date

weekly_returns

  • id · week_id → weeks
  • catering_turnover, tgtg_turnover
  • catering_food_cost
  • logistics_cost, num_drops
  • unsold_items, remarks
  • manual inputs only — the rest is derived

sales_facts

  • id · week_id → weeks
  • grain fridge|supplier|category
  • entity_id → per grain
  • total_items, sales_turnover
  • refund, fl_discount, customer_credit

dispatch_facts

  • id · week_id → weeks
  • grain + entity_id
  • foodcost_value numeric

added_facts

  • id · week_id → weeks
  • grain + entity_id
  • added_value numeric

fridge_fractions

  • fridge_id → fridges
  • month date (1st)
  • fraction numeric

service_additionals

  • fridge_id → fridges
  • month date (1st)
  • amount numeric

import_runs

  • id · source, scope
  • row_count, status
  • unmapped_names jsonb
  • created_at timestamptz

v_weekly_kpis (view)

  • turnover ex VAT, POS fee, RFID fee
  • food margin €/%, per-fridge turnover
  • logistics %, net margin €/%
  • = all 12 “Calculated Fields” columns

v_monthly_client_pnl (view)

  • per fridge × month
  • food cost, turnover, food margin
  • fraction logistics, POS & SW
  • client fee /12, net margin

v_monthly_supplier_pnl · v_monthly_category_pnl

  • per supplier / category × month
  • food cost, turnover, margin
  • RFID fees, net margin
